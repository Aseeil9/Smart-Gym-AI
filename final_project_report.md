# 🏋️ تقرير مشروع Ai.sport - النظام الرياضي الذكي
## تحليل الأداء والتحليلات المتقدمة بالذكاء الاصطناعي

---

## 1. نظرة عامة على المشروع
مشروع **Ai.sport** هو منصة رياضية متكاملة تهدف إلى رقمنة تجربة التدريب داخل الصالات الرياضية باستخدام تقنيات الذكاء الاصطناعي وإنترنت الأشياء (IoT). يوفر النظام بوابة تجمع بين الرياضي، المدرب، والإدارة في بيئة واحدة متزامنة لحظياً وذكية.

## 2. المميزات الرئيسية للنظام

### 🏃 بوابة الرياضي (Athlete Portal)
*   **مراقبة العلامات الحيوية:** عرض نبض القلب والخطوات والسعرات المحروقة لحظياً عبر رسوم بيانية تفاعلية.
*   **نظام تتبع النوم الذكي:** إمكانية تسجيل ساعات النوم والحصول على تحليل جودة النوم.
*   **توصيات AI اليومية:** يقدم النظام نصائح مخصصة بناءً على جودة نوم الرياضي (مثلاً: زيادة الجهد عند النوم الجيد، أو تقليله لتجنب الإجهاد).
*   **نداء الطوارئ (SOS):** زر مخصص لإرسال تنبيه استغاثة فوري لجميع المدربين في حال الشعور بالتعب أو الإصابة.
*   **قائمة المدربين:** استعراض المدربين المتواجدين في النادي وساعات عملهم المتاحة.

### 📋 بوابة المدرب (Coach Dashboard)
*   **تحليلات الفريق الحية:** بطاقات إحصائية تظهر عدد الأنشط، متوسط نبض الفريق، وإجمالي الخطوات المحققة.
*   **نظام التنبيهات الحرجة:** رصد آلي لأي رياضي يتجاوز معدل نبضه الحدود الآمنة، مع عرض نسبة خطورة الإصابة المقدرة بـ AI.
*   **إدارة النصائح العامة:** إمكانية كتابة نصيحة عامة لجميع الرياضيين (نصيحة اليوم) وتحديث ساعات التواجد.
*   **التفاعل الفوري:** زر لإرسال تنبيهات مباشرة للرياضيين الذين يمرون بحالات إجهاد.

### ⚙️ بوابة الإدارة (Admin Panel)
*   **إدارة الاشتراكات:** ربط المستخدمين بالنادي وتحديد تواريخ انتهاء العضويات.
*   **إعادة تدريب النماذج:** نظام "التعلم المستمر" الذي يسمح للمسؤول بإعادة تدريب خوارزميات الذكاء الاصطناعي بناءً على البيانات الجديدة المجمعة.

## 3. المعمارية التقنية (Tech Stack)

| المكون | التقنية المستخدمة |
| :--- | :--- |
| **Backend (الباك إند)** | FastAPI (Python) - لضمان السرعة العالية في معالجة طلبات الـ API. |
| **Frontend (الواجهة)** | HTML5, CSS3 (Vanilla), JavaScript (ES6+). |
| **Database (قاعدة البيانات)** | SQLite (SQLAlchemy ORM) - لسهولة تداول المشروع وقوته في التعامل مع العلاقات. |
| **AI/ML Engine** | Scikit-learn & Linear Regression - لتحليل مخاطر الإصابة وتوليد الخطط. |
| **Real-time Sync** | WebSockets - لضمان وصول نداءات الاستغاثة والتنبيهات في جزء من الثانية. |
| **Visualizations** | Chart.js - لعرض الرسوم البيانية للنبض والخطوات وحالة النوم. |

## 4. هيكل قاعدة البيانات (Schema)
تم تصميم قاعدة البيانات بعدة جداول مترابطة لضمان تكامل البيانات:
1.  **Users:** تخزين بيانات الرياضيين والمدربين وصلاحياتهم.
2.  **SensorData:** الاحتفاظ بسجل العلامات الحيوية (نبض، خطوات، سعرات).
3.  **SleepData:** سجلات النوم اليومية.
4.  **AIPredictions:** نتائج تحليل الذكاء الاصطناعي لخطورة الإصابة.
5.  **GymMemberships:** إدارة العضويات والاشتراكات.

## 5. نظام التنبؤ الذكي (AI Logic)
يعتمد النظام على منطق احتمالي (Probabilistic Logic) لتحليل البيانات:
*   إذا انخفض معدل النوم وزاد معدل النبض أثناء التمرين -> النظام يرفع مستوى الخطر ويرسل نداء SOS آلي للمدرب.
*   يتم تعديل أهداف الخطوات والسعرات ديناميكياً بناءً على "معامل التعافي" المستخلص من ساعات النوم.

## 6. الخاتمة والتطوير المستقبلي
يعد **Ai.sport** نموذجاً متقدماً للصالات الرياضية الحديثة. تكمن القوة في تجميع البيانات وتحويلها إلى قرارات ذكية تحفظ سلامة الرياضي وتساعد المدرب على إدارة الفريق بكفاءة أعلى.

**التطلعات المستقبلية:**
1.  ربط النظام بحساسات حقيقية (عبر بروتوكول MQTT).
2.  تطبيق التعرف على الوجه (Face ID) لتسجيل الحضور التلقائي.
3.  بناء تطبيق جوال (Flutter) مرتبط بنفس الباك إند.
